# 接口设计

> 本文档定义课程助手后端的 REST API 规范。  
> 数据格式：JSON（`Content-Type: application/json`）  
> 字符编码：UTF-8

---

## 接口总览

| 方法 | 路径 | 功能 | 认证 |
|------|------|------|------|
| GET | `/health` | 服务健康检查 | 无 |
| POST | `/chat` | 课程问答 | 可选 Token |
| POST | `/review` | 作业批改 | 可选 Token |
| POST | `/search` | 知识检索（RAG 阶段） | 可选 Token |

---

## 1. GET /health

### 功能

返回服务当前运行状态，供负载均衡、K8s 探针和监控系统使用。

### 请求

无请求体。

### 响应

**正常**（HTTP 200）：

```json
{
  "status": "ok",
  "version": "0.1.0",
  "uptime_seconds": 3600
}
```

**降级**（HTTP 200，内部依赖异常）：

```json
{
  "status": "degraded",
  "details": "LLM API unreachable"
}
```

## 2. POST /chat

### 功能

接收学生提问，调用 LLM 生成教学友好的回答。

### 请求体

```json
{
  "question": "什么是递归？",
  "session_id": "optional-session-uuid",
  "options": {
    "temperature": 0.7,
    "max_tokens": 512
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `question` | string | ✅ | 学生问题，最大 4096 字符 |
| `session_id` | string | ❌ | 会话 ID，用于多轮对话 |
| `options.temperature` | float | ❌ | 模型温度，默认 0.7 |
| `options.max_tokens` | int | ❌ | 最大生成 token 数，默认 512 |

### 响应

**成功**（HTTP 200）：

```json
{
  "answer": "递归（Recursion）是指函数直接或间接调用自身的编程技术...",
  "session_id": "session-uuid",
  "tokens_used": 128,
  "model": "gpt-4o-mini"
}
```

**错误**（HTTP 400）：

```json
{
  "error": "invalid_request",
  "message": "question 字段不能为空"
}
```

**服务错误**（HTTP 503）：

```json
{
  "error": "llm_unavailable",
  "message": "大模型服务暂时不可用，请稍后重试"
}
```

### C++ 请求体解析示例

```cpp
#include <nlohmann/json.hpp>
using json = nlohmann::json;

void handleChat(const std::string& requestBody) {
    try {
        auto body = json::parse(requestBody);
        std::string question = body.at("question").get<std::string>();
        // 可选字段用 value()
        float temp = body.value("temperature", 0.7f);
    } catch (const json::exception& e) {
        // 返回 400 错误
    }
}
```

## 3. POST /review

### 功能

对学生提交的代码或文本进行规则校验，返回评分和改进建议。

### 请求体

```json
{
  "code": "int main() { int* p = new int(1); return 0; }",
  "language": "cpp",
  "assignment_id": "hw-03"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `code` | string | ✅ | 提交的代码文本，最大 16384 字符 |
| `language` | string | ✅ | 编程语言（`cpp` / `python`） |
| `assignment_id` | string | ❌ | 作业 ID，用于关联评分规则 |

### 响应

**成功**（HTTP 200）：

```json
{
  "status": "ok",
  "score": 75,
  "max_score": 100,
  "comment": "代码结构清晰，但存在内存泄漏（new 而无 delete）",
  "issues": [
    {
      "type": "memory_leak",
      "line": 1,
      "message": "分配了内存但未释放，建议使用 std::unique_ptr"
    }
  ]
}
```

**空提交**（HTTP 400）：

```json
{
  "error": "empty_submission",
  "message": "提交内容不能为空"
}
```

## 4. POST /search（RAG 阶段，预留接口）

### 功能

基于向量检索从课程知识库中找出最相关的文档片段。

### 请求体

```json
{
```

### 响应

```json
{
  "results": [
    {
      "content": "虚函数通过 vtable 实现多态...",
      "source": "第3章_多态与虚函数.md",
      "score": 0.92
    }
  ]
}
```

## 通用错误码

| HTTP 状态码 | error 字段 | 说明 |
|------------|-----------|------|
| 400 | `invalid_request` | 请求参数非法 |
| 400 | `empty_submission` | 必填字段为空 |
| 401 | `unauthorized` | 缺少有效认证 |
| 429 | `rate_limited` | 请求频率超限 |
| 503 | `llm_unavailable` | LLM 服务不可用 |
| 500 | `internal_error` | 服务内部错误 |

## C++ 响应构造示例

```cpp
// 使用 nlohmann/json 构造成功响应
nlohmann::json resp;
resp["answer"] = answer;
resp["tokens_used"] = tokensUsed;
return resp.dump();  // 序列化为字符串

// 使用 fmt 构造简单响应（无需引入 json 库）
return fmt::format(R"({{"status":"ok","answer":"{}"}})", answer);
```
