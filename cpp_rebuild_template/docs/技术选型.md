# 技术选型

> 本文档对比各技术选项的优劣，帮助学习者根据自身情况做出合理选择。  
> **原则**：优先选择文档完善、社区活跃、学习曲线平缓的库。

---

## 1. HTTP 框架对比

本模板通过 `server::HttpServer` 抽象接口隔离 HTTP 框架，**可随时替换底层实现**。

| 维度 | [Crow](https://github.com/CrowCpp/Crow) | [oat++](https://oatpp.io/) | [Drogon](https://github.com/drogonframework/drogon) |
|------|---------|--------|---------|
| 上手难度 | ⭐ 最低 | ⭐⭐ 中等 | ⭐⭐⭐ 较高 |
| 依赖方式 | Header-only | 需编译安装 | 需编译安装 |
| 性能 | 中等 | 高 | 非常高 |
| 异步支持 | 基础 | 完整 | 完整（协程） |
| WebSocket | ✅ | ✅ | ✅ |
| 中间件 | ✅ | ✅ | ✅ |
| ORM | ❌ | ❌ | ✅ |
| 文档质量 | 好 | 好 | 中等 |
| 学习项目推荐 | **首选** | 第二选 | 进阶选 |

**推荐路径**：

```
Crow（入门，header-only，1小时内跑通）
  ↓ 熟悉后
oat++（中等规模项目）
  ↓ 有性能需求时
Drogon（生产级，协程/ORM）
```

**Crow 快速接入示例**：

```cpp
#include "crow.h"
int main() {
    crow::SimpleApp app;
    CROW_ROUTE(app, "/health")([]{ return R"({"status":"ok"})"; });
    app.port(8080).run();
}
```

## 2. JSON 库

| 库 | 推荐指数 | 特点 |
|----|---------|------|
| [nlohmann/json](https://github.com/nlohmann/json) | ⭐⭐⭐⭐⭐ | Header-only，语法直观，事实标准 |
| [RapidJSON](https://github.com/Tencent/rapidjson) | ⭐⭐⭐ | 性能最优，但 API 较繁琐 |
| [simdjson](https://github.com/simdjson/simdjson) | ⭐⭐⭐ | 极高性能，适合大吞吐量场景 |

**结论**：学习阶段统一用 `nlohmann/json`，生产优化时再考虑 `simdjson`。

```cmake
# CMakeLists.txt 中引入
FetchContent_Declare(nlohmann_json
    GIT_REPOSITORY https://github.com/nlohmann/json.git
    GIT_TAG        v3.11.3)
FetchContent_MakeAvailable(nlohmann_json)
target_link_libraries(your_target PRIVATE nlohmann_json::nlohmann_json)
```

## 3. 日志库

| 库 | 推荐指数 | 特点 |
|----|---------|------|
| [spdlog](https://github.com/gabime/spdlog) | ⭐⭐⭐⭐⭐ | 高性能、异步、多 sink、header-only 可选 |
| [glog](https://github.com/google/glog) | ⭐⭐⭐ | Google 出品，功能完善但稍重 |
| `std::cout` | ❌ | 仅用于 demo，生产禁用 |

**结论**：统一用 `spdlog`。

```cpp
#include "spdlog/spdlog.h"
spdlog::info("请求开始 question={}", question);
spdlog::error("LLM 调用失败: {}", errorMsg);
```

## 4. 格式化库

| 库 | 推荐指数 | 说明 |
|----|---------|------|
| [fmt](https://github.com/fmtlib/fmt) | ⭐⭐⭐⭐⭐ | `std::format` 的前身，C++20 已纳入标准 |
| `std::ostringstream` | ⭐⭐ | 无需额外依赖，但性能和可读性差 |
| `sprintf` | ❌ | 类型不安全，禁用 |

```cpp
// fmt 示例
#include "fmt/format.h"
std::string msg = fmt::format("学生 {} 提问: {}", studentId, question);
```

## 5. 测试框架

| 框架 | 推荐指数 | 特点 |
|------|---------|------|
| [GoogleTest](https://github.com/google/googletest) | ⭐⭐⭐⭐⭐ | Google 出品，功能完善，与 CMake 深度集成 |
| [Catch2](https://github.com/catchorg/Catch2) | ⭐⭐⭐⭐ | Header-only，BDD 风格，上手更快 |
| [doctest](https://github.com/doctest/doctest) | ⭐⭐⭐ | 超轻量，适合嵌入式 |

**结论**：本模板使用 `GoogleTest`；如更偏好简洁语法可改用 `Catch2`。

## 6. HTTP 客户端（调用 LLM API）

| 库 | 推荐指数 | 特点 |
|----|---------|------|
| [cpr](https://github.com/libcpr/cpr) | ⭐⭐⭐⭐⭐ | libcurl 的现代 C++ 封装，API 直观 |
| libcurl | ⭐⭐⭐ | 功能最全，但 C 风格 API 繁琐 |
| Boost.Beast | ⭐⭐ | Boost 生态，学习曲线陡峭 |

```cmake
FetchContent_Declare(cpr
    GIT_REPOSITORY https://github.com/libcpr/cpr.git
    GIT_TAG        1.10.5)
FetchContent_MakeAvailable(cpr)
target_link_libraries(llm_lib PRIVATE cpr::cpr)
```

## 7. 依赖管理策略

本模板使用 CMake `FetchContent` 自动下载依赖，**无需手动安装**。

```cmake
include(FetchContent)
FetchContent_Declare(googletest ...)
FetchContent_MakeAvailable(googletest)
```

进阶阶段可切换到 [vcpkg](https://vcpkg.io/) 或 [Conan](https://conan.io/) 进行系统级包管理。

## 8. 升级到 C++20

修改 `CMakeLists.txt`：

```cmake
set(CMAKE_CXX_STANDARD 20)
```

可逐步引入的 C++20 特性：

| 特性 | 场景 |
|------|------|
| `std::format` | 替换 `fmt::format`（标准化） |
| `concepts` | 模板约束，提升错误信息可读性 |
| `ranges` | 替换手写循环 |
| `coroutines` | 异步 I/O（配合 Drogon/Asio） |
| `std::jthread` | 自动 join 的线程，替代 `std::thread` |
