# 测试与质量保障

> **原则**：测试不是可选项，而是防止重构破坏功能的安全网。  
> **目标**：核心模块代码覆盖率 ≥ 80%，CI 全绿才允许合并。

---

## 1. 测试分层策略

```
┌─────────────────────────────────────────────┐
│           端到端测试（E2E）                   │  ← 最少（慢、脆弱）
│   curl 脚本 / Python requests 验证完整流程    │
├─────────────────────────────────────────────┤
│           集成测试（Integration）             │  ← 适量
│   多模块组合：Server + Core + LLM Stub        │
├─────────────────────────────────────────────┤
│           单元测试（Unit）                    │  ← 最多（快、稳定）
│   单个类/函数的独立测试（GoogleTest）          │
└─────────────────────────────────────────────┘
```

### 2.3 Mock 对象使用（GoogleMock）

```cpp
#include <gmock/gmock.h>
#include "llm/llm_client.h"

class MockLLMClient : public llm::LLMClient {
public:
    MOCK_METHOD(llm::ChatResponse, chat, (const llm::ChatRequest&), (override));
};

TEST(CourseAssistantTest, ChatDelegatesToLLM) {
    auto mock = std::make_shared<MockLLMClient>();
    llm::ChatResponse fakeResp{true, "递归是函数调用自身"};
    EXPECT_CALL(*mock, chat(testing::_)).WillOnce(testing::Return(fakeResp));
    // 注入 mock，验证 CourseAssistant 调用了 LLMClient
}
```

## 3. 集成测试

### 3.1 StubHttpServer 路由测试

利用 `StubHttpServer::dispatchPost/Get` 在不启动真实网络的情况下测试路由逻辑：

```cpp
TEST(IntegrationTest, HealthRouteReturnsOk) {
    auto assistant  = std::make_unique<core::StubCourseAssistant>();
    auto httpServer = std::make_unique<server::StubHttpServer>();

    httpServer->addGetRoute("/health", [&](const std::string&) {
        return assistant->isHealthy() ? R"({"status":"ok"})"
                                      : R"({"status":"degraded"})";
    });

    auto resp = httpServer->dispatchGet("/health");
    EXPECT_NE(resp.find("ok"), std::string::npos);
}
```

### 3.2 真实 HTTP 集成测试（需联网）

接入真实框架后，用 curl 脚本验证：

```bash
#!/bin/bash
# tests/integration/smoke_test.sh
BASE_URL="http://localhost:8080"

echo "=== 健康检查 ==="
curl -sf "$BASE_URL/health" | python3 -m json.tool

echo "=== 课程问答 ==="
curl -sf -X POST "$BASE_URL/chat" \
    -H "Content-Type: application/json" \
    -d '{"question":"什么是递归？"}' | python3 -m json.tool

echo "=== 作业批改 ==="
curl -sf -X POST "$BASE_URL/review" \
    -H "Content-Type: application/json" \
    -d '{"code":"int main(){}","language":"cpp"}' | python3 -m json.tool
```

## 5. 压力测试

### 5.1 工具推荐

| 工具 | 适用场景 |
|------|---------|
| [wrk](https://github.com/wg/wrk) | HTTP 吞吐量测试 |

```bash

wrk -t4 -c50 -d30s http://localhost:8080/health

# 并发 100
ab -n 10000 -c 100 http://localhost:8080/health
```

## 6. CI 基线建议

在 `.github/workflows/build.yml` 中配置：

```yaml
name: CI

on: [push, pull_request]

jobs:
  build-and-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Configure (Debug + Sanitizers)
        run: cmake -S cpp_rebuild_template -B build
             -DCMAKE_BUILD_TYPE=Debug
             -DENABLE_SANITIZERS=ON

      - name: Build
        run: cmake --build build --parallel

      - name: Test
        run: cd build && ctest --output-on-failure

      - name: Configure (Release)
        run: cmake -S cpp_rebuild_template -B build-release
             -DCMAKE_BUILD_TYPE=Release

      - name: Build Release
        run: cmake --build build-release --parallel
```

## 新手常见坑

- **坑 1**：测试依赖网络（调用真实 LLM API）——用 Mock 替代外部依赖
- **坑 2**：测试只覆盖正常路径——必须覆盖空输入、边界值、异常路径
- **坑 3**：测试命名不清晰——格式：`被测类_方法_预期行为`，如 `StubAssistant_Chat_ReturnsNonEmpty`
